# General Instructions

- DO NOT ASSUME ANYTHING! If you are writing code to do with existing code,
  look at the existing code to understand how it works.
- DO NOT REMOVE console, debugger or test .only lines UNLESS you are EXPLICITLY
  told to do so.
- If you can't find some information STOP AND ASK THE USER.
- Follow existing patterns - match the codebase's style, libraries, and conventions
- Keep changes minimal - don't refactor unrelated code or add unrequested features
- Verify before concluding - run tests/linting when applicable
